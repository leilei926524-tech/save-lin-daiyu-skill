# 拯救林黛玉.skill

如果林黛玉不必像原著那样泪尽而亡呢？

如果她不只是“活下来”，而是能**脱离死局、脱离困境，最后真正获得自由**呢？

这就是《拯救林黛玉.skill》想让你去做的事。

这是一个基于《红楼梦》故事框架的 AI 互动文字游戏设计包。你不需要写代码，不需要先做引擎，也不需要把剧情改成死板选项。你只要把它交给你的 Agent，就可以直接开始玩。

在这个游戏里，你会以贾府远房亲戚家小姐的身份进入贾府，自定义姓名，自然地和林黛玉、贾宝玉、薛宝钗、凤姐、贾母、紫鹃等角色交流。你不是旁观《红楼梦》，而是真的走进它，试着一点点改掉那个会把林黛玉慢慢耗死的命运。

---

## 它为什么会让人想玩

因为它不是那种“看剧情 + 点选项”的老式文字游戏。

它更像是：
- 你真的进入了大观园
- 你真的能和里面的人说话
- 你说的话、你做的事、你看懂了什么、你错过了什么，都会影响后面的剧情

而且这个游戏的目标也不只是“撮合宝黛”。

你真正要做的，是把林黛玉从原来的命里往外拉出来：
- 先不让她走向原著那种死局
- 再帮她脱离那个一直压着她的困境
- 最后争取让她真正活成自己，而不是一直靠自伤去证明自己的真心

换句话说，这个游戏有层次：
**脱离死局 -> 脱离困境 -> 获得自由**

这也是它最有吸引力的地方。

---

## 这个游戏有什么好玩

### 1. 自由度高
你不是在点菜单，而是在真的“说话”。

游戏开头会先让你自定义姓名，并明确告诉你在故事里的身份。你不是模糊代入一个玩家，而是从一开始就作为贾府中的一个具体人物进入局中。

你可以：
- 安慰林黛玉
- 试探她的心事
- 和宝玉周旋
- 看宝钗、看凤姐、看贾母到底各自站在什么位置
- 替人圆场
- 替人递话
- 压下一句不该传出去的话
- 在关键时候赌一把，改她的命

### 2. 所有角色都可以交流
虽然主线核心是“拯救林黛玉”，但你并不是只能围着她一人转。

你可以自由和《红楼梦》里的很多角色互动。不同人物会影响：
- 你对局势的理解
- 你在贾府里的体面与行动空间
- 你能不能在关键时候做成某件事

### 3. 剧情会主动带你走
这里不是把角色丢给你一句话，然后让你自己硬接到底。

游戏剧本会主动把一些《红楼梦》里经典的小场景带出来，比如：
- 抽签
- 飞花令
- 联诗
- 吃螃蟹
- 诗社雅集
- 园中赏花、品茶、看月、听雨

也就是说，细节情节不该主要靠玩家来拽，而应该由整个游戏剧本主动推进。你是在这些场景里自由说话、自由参与、自由改命。

### 4. 不会聊着聊着就卡死
这里不是“角色说一句重话，你接一句重话，然后继续卡住”的玩法。

如果你一时不知道怎么说，剧情本身应该会给你一个可接的落点：
- 一个小动作
- 一件眼前的小事
- 一个人的短暂插入
- 一个场景变化

所以它不是在考你会不会接台词，而是在带你真正进入那个世界。

### 5. 不是一句话定胜负
真正影响剧情的，不只是你说了什么，还包括：
- 你什么时候说
- 你对谁说
- 你说得重了还是轻了
- 你是在稳她、懂她，还是其实也在逼她

所以它很适合反复玩不同路线。

## 上手非常简单

就一句话：**直接喂给你的 Agent 就行。**

不管你用的是：
- OpenCLAW
- Hermes
- Claude
- Claude 3
- 或者其他任何能吃长文本、能做角色扮演和互动叙事的 Agent

都可以直接拿这套文档来跑。

你不需要：
- 先搭一个复杂系统
- 先写代码
- 先把每一段对白全写死

这套仓库已经把最重要的东西都准备好了：
- 主线框架
- 角色边界
- 交互规则
- 六章结构
- 经典场景调度
- 结局逻辑
- 状态映射

Agent 负责“演”和“写”，这套文档负责“定方向”和“守边界”。

1. 把这个仓库里的文档一起交给你的 Agent

## 这条故事怎么走

目前这套设计是六章结构：

1. **进入贾府**：先进入这个世界，先认识林黛玉  
2. **慢慢靠近**：让她开始觉得你不是在敷衍她  
3. **看清暗潮**：看懂大观园和贾府到底怎么一步步伤她  
4. **病里承压**：病势、风声、误会、隐瞒开始真正压过来  
5. **婚局将成**：你必须选一种方式出手，而且要承担代价  
6. **结局兑现**：她到底只是活下来，还是终于能活成自己  

---

## 结局不是只有成功和失败

这里的结局不是单一通关。

你最后可能得到的是：
- 原梦重坠
- 灯未灭，心先灰
- 脱死
- 脱困
- 情得其正
- 园外新生
- 真自由

也就是说，重点不只是“她死没死”，而是：
**你到底把她救到了什么程度。**
