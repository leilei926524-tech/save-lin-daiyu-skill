# save-lin-daiyu-skill

《拯救林黛玉》是一个面向 Agent 的 AI 互动文字游戏设计包。

它以《红楼梦》为底本，不追求把所有对白逐字写死，而是提供一套可供 Claude、Gemini、MiniMax、Kimi 等模型调用的结构化框架：人物边界、交互规则、章节主线、状态映射与结局逻辑。

这个项目的核心命题不是“怎样让林黛玉嫁给贾宝玉”，而是：

**怎样让林黛玉不再被寄居、误会、无名之情与自我耗损逼至绝路。**

---

## 项目定位

这是一个 **Agent-first** 的 skill 设计包。

也就是说，这套文档的主要用途不是充当逐字剧本，而是为已经具备语言生成能力的 Agent 提供：

- 稳定的主线框架
- 明确的人物边界
- 自然语言交互规则
- 六章式命运推进结构
- 全局状态与结局映射

具体叙述、对白、铺陈和单轮渲染，可交给接入该 skill 的模型自行生成。

---

## 适用场景

这套设计包适合用于：

- Claude Code / Claude Agent SDK skill 设计
- 其他 Agent 系统的 narrative skill / story skill
- 自然语言驱动的互动叙事原型
- 《红楼梦》题材的 AI 文字游戏框架搭建

---

## 核心特点

### 1. 自然语言驱动
玩家主要通过自由输入推进故事，而不是依赖固定选项。

### 2. 表面自由，底层有判定
系统在后台判断玩家输入的：
- 意图
- 分寸
- 时机
- 风险

并通过人物反应和叙事变化体现后果。

### 3. 不是单纯恋爱线
项目重点不是“宝黛配对”，而是改写林黛玉的命运结构。

### 4. 面向 Agent，而非写死剧本
文档优先提供框架与运行逻辑，而不是写满每一段具体文案。

## 六章结构概览

1. **第一章：进入**  
   让玩家进入贾府，初识林黛玉。

2. **第二章：靠近**  
   在日常往来中真正接近她，触碰寄居感与宝黛情势。

3. **第三章：看清**  
   看清大观园与贾府秩序如何一步步偏向“更稳妥”的人。

4. **第四章：承压**  
   病势、风声、误传、错位开始真正压到黛玉身上。

5. **第五章：出手与锁线**  
   婚局将成，玩家必须选择主救援策略，并承担代价。

6. **第六章：兑现与结局**  
   回答一个最终问题：即便命救下来，她以后怎么活。

## 状态系统

本项目不是单一好感度游戏，而是多轴状态系统。

主要状态包括：
- 黛玉信任
- 黛玉心防
- 黛玉病势
- 宝黛情势
- 主角声望
- 疑影
- 安置基础（隐含状态）

这些状态共同决定不同章节的可行动空间，以及最终结局的层级。

详见：
- [01_交互规则.md](01_交互规则.md)
- [12_全局状态与结局映射.md](12_全局状态与结局映射.md)

## 结局方向

当前设计包含以下层级：

- 原梦重坠
- 灯未灭，心先灰
- 脱死
- 脱困
- 情得其正
- 园外新生
- 真自由

重点不是只看她是否活下来，而是看她是否真正摆脱那条会慢慢耗死她的命轨。

## 使用原则

在接入任意 Agent 时，建议遵守以下原则：

- 不把这套文档当成逐字照读脚本
- 不把自然语言交互退化成关键词猜谜
- 不把林黛玉写成单薄恋爱对象
- 不把薛宝钗等人物粗暴黑化
- 不追求表面爽感而牺牲人物复杂性
