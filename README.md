# 拯救林黛玉.skill

这是一个基于《红楼梦》故事框架的 AI 文字游戏设计包。

你可以把它理解成一个已经搭好骨架的互动故事世界：
玩家进入贾府之后，不是只能点选项，也不是只能围着固定剧情走，而是可以**自由地和《红楼梦》里的角色交流**，试着靠自己的判断、话术、试探、安慰、布局与取舍，去改变林黛玉的命运。

它的核心乐趣，不是照着标准答案通关，而是：

- 你能不能真的接住林黛玉
- 你能不能看懂贾府里那些没说出口的暗流
- 你能不能在不惊动全局的情况下，一点点把她从原来的命数里救出来

---

## 这是什么

《拯救林黛玉》不是一本写死的小说，也不是那种只能选 A/B/C/D 的传统文字游戏。

它更像是一个**可以交给 Agent 直接运行的互动叙事框架**：

- 故事底本是《红楼梦》
- 玩法是自然语言对话
- 玩家可以自由开口
- Agent 负责根据你的话，推动人物反应、局势变化和剧情走向

你说的话，不只是“台词”，而是真的会影响：
- 林黛玉会不会信你
- 她会不会把真心话告诉你
- 你会不会被别人怀疑
- 宝玉、宝钗、贾母、凤姐等人会怎样看你
- 后面的婚局、病势、误会和结局会怎样变化

---

## 这个游戏有什么好玩的

### 1. 自由度高
你不是在点菜单，而是在真的“说话”。

游戏开头还会先让你自定义姓名，并明确告诉你在故事里的身份。也就是说，你不是模糊代入一个“玩家”，而是从一开始就作为贾府中的一个具体人物进入局中。

你可以：
- 安慰林黛玉
- 试探她的心事
- 和宝玉周旋
- 观察宝钗
- 向凤姐套话
- 对贾母装乖
- 替谁圆场
- 替谁递话
- 压下一句不该传出去的话
- 在关键时刻赌一把，改她的命

### 2. 不只是恋爱线
这个游戏不是简单的“帮黛玉和宝玉在一起”。

真正要改的是她整个人生的走向：
- 她为什么会一步步被耗垮
- 她到底缺的是什么
- 她怎样才能不再困在原来的命里

### 3. 所有角色都可以交流
虽然主线核心是“拯救林黛玉”，但玩家并不是只能和她说话。

你可以和贾府、大观园里的其他角色互动。不同角色会影响你对局势的理解，也会影响你有没有机会把一些关键事情做成。

### 4. 不会聊着聊着就卡死
这里不是把角色丢给你一句话，然后让你自己硬接到底。

Agent 应该会稍微带着剧情走：
- 对话里会穿插一些小剧情
- 会有场面变化、人物进出、细节落点
- 当你一时不知道说什么时，剧情会自然给你一个可接的方向

### 5. 不是一句话定胜负
这里没有“猜关键词”式玩法。

真正影响剧情的，是：
- 你说了什么
- 你什么时候说
- 你对谁说
- 你说得重了还是轻了
- 你是懂她，还是只是会安慰

所以可玩性很高，也很适合反复跑不同路线。

## 怎么用

很简单：**直接喂给你的 Agent 就行。**

不管你用的是：
- OpenCLAW
- Hermes
- Claude
- Claude 3
- 或者其他任何能吃长文本、能做角色扮演和互动叙事的 Agent

都可以直接拿这套文档来跑。

你不需要先把它改成代码，也不需要先把每一段对白都写死。

这套仓库本身已经把最重要的东西搭好了：
- 主线框架
- 角色边界
- 交互规则
- 六章结构
- 结局逻辑
- 状态映射

Agent 本身负责“演”和“写”，这套文档负责“定方向”和“守边界”。

## 建议使用方式

最简单的办法是：

1. 把这个仓库里的文档一起交给你的 Agent
2. 让它读取项目总纲、交互规则、角色卡、章节设计和状态映射
3. 开局先让玩家自定义姓名，并告知玩家在故事中的身份
4. 然后从第一章开始跑
5. 玩家用自然语言和角色交流
6. Agent 根据这套框架往下推进剧情

## 故事怎么走

这套设计目前是六章结构：

1. **进入贾府**：先进入这个世界，先认识林黛玉
2. **慢慢靠近**：让她开始觉得你不是在敷衍她
3. **看清暗潮**：看懂大观园和贾府到底怎么一步步伤她
4. **病里承压**：病势、风声、误会、隐瞒开始真正压过来
5. **婚局将成**：你必须选一种方式出手，而且要承担代价
6. **结局兑现**：她到底只是活下来，还是终于能活成自己

---

## 结局不是只有“成功 / 失败”

这里的结局不是单一通关。

你最后可能得到的是：
- 原梦重坠
- 灯未灭，心先灰
- 脱死
- 脱困
- 情得其正
- 园外新生
- 真自由

也就是说，重点不只是“她死没死”，而是：
**你到底把她救到了什么程度。**

- 你想围绕林黛玉做一条真正有层次的改命线
