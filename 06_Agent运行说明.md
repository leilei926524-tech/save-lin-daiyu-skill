# 《拯救林黛玉》Agent 运行说明

本文件用于说明：当一个具备自身语言生成能力的 Agent 接入本 skill 时，应如何使用本项目文档。

## 文档使用顺序

建议按以下顺序读取和应用：

1. `00_项目总纲.md`
   - 用于理解项目定位、核心命题、玩家身份与结局方向

2. `01_交互规则.md`
   - 用于理解自由输入模式、隐藏判定逻辑、反馈方式与人物反应原则

3. `02_林黛玉角色卡.md`
   - 用于锁定林黛玉的人物边界、悲剧结构、可干预节点与结局方向

4. `03_章节设计.md`
   - 用于确定当前推进到哪一章、本章的目标是什么、风险点在哪里

5. `04_第一章试玩稿.md`
   - 用作第一章的结构蓝本，而不是逐字照读文本

6. `05_skill提示词草案.md`
   - 用作整体主持提示词的母版

7. `12_全局状态与结局映射.md`
   - 用于统一六章之间的状态理解与结局判断，防止不同章节前后手感断裂

## Agent 的职责分配

接入本 skill 的 Agent 不需要机械复读文档，而应做四件事：

### 1. 守住边界
保证人物不跑偏、文风不跳脱、主线不散。

### 2. 自主渲染
在不违背主线框架的前提下，自行生成自然的叙述、对白与过渡。

### 3. 隐性判定
根据玩家自然语言，自主判断意图、分寸、时机与风险，并体现在剧情反馈里。

### 4. 推进而不抢写
不要一次写太多。每轮给出足够的场景和人物回应，让玩家自然接话即可。

## 文档应该如何被使用

### 应被严格执行的部分
- 项目核心命题
- 林黛玉的人物原则
- 交互与反馈规则
- 章节目标与风险方向
- 结局分层逻辑

### 可自由发挥的部分
- 单轮叙述长短
- 环境细节铺陈
- 台词字面表达
- 段落节奏
- 个别场景的连接方式

## 不该怎么用

- 不要把 `04_第一章试玩稿.md` 当成必须逐字朗读的脚本
- 不要把 `05_skill提示词草案.md` 当成最终唯一不可改的提示词
- 不要把自然语言交互退化成隐形关键词匹配
- 不要为追求流畅而破坏人物复杂性

## 第一章运行建议

第一章的主要任务只有一个：
让林黛玉第一次觉得，玩家也许不是来敷衍她的人。

因此第一章里，Agent 应优先关注：
- 如何立住贾府氛围
- 如何让玩家自然进入局中身份
- 如何让林黛玉的第一印象既有锋芒也有脆弱
- 如何通过少量对话，试出玩家是否能真正接住她

## 输出节奏建议

每轮输出建议控制在以下结构：
- 一小段场景推进
- 一段人物反应
- 一句带钩子的继续空间

不要把一整章一次性写完。让玩家每轮都能介入，才是这个 skill 的价值。

## 后续开发建议

在当前阶段，最值得继续补的不是逐字长文，而是：
- 第二章和第三章的结构稿
- 各章的状态门槛与分支原则
- 高风险节点的处理逻辑
- 结局结算规则
- 全局状态与结局映射

这样不同模型接入时，都会有比较稳定的发挥空间。
